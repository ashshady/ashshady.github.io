STL使用总结

四月 12, 2021 发布在 pat, stl

自己总结的一些STL使用方法，迭代学习，随时进步

emm使用STL也有一段时间，但都是偶尔查查博客一些零碎的用法，没有系统的看过，没有系统的学习过，看了晴神的算法笔记决定写一篇博客总结一下，当然在以后的使用中遇见其他的用法也会记录下来，迭代加递归学习吧，keep learning

1. vector：可变长数组

#include<vector> using namespace std;  // 头文件

声明：vector v1；typename可以是基本数据类型，结构体和其他STL容器，如果是STL容器，在C++11之前要在末尾的»中间加一个空格，否则会识别成移位操作。

vector是可变长的数组，也可声明成二维的定长/变长数组：vector v1[100]；这样在一维上是定长的，在二维上是变长的；甚至还可以：vector<vector> v1；这样在两个维度上都是变长的

vector的访问：下标和迭代器

  • 可以像数组一样通过下标访问：vector[i]；

  • 迭代器：vector::iterator it；其实C++11之后可以直接写成：auto it，这样可以自动识别类型（多在for中直接写，既安全又简便），用*it访问里面的元素。迭代器支持自加和自减操作（it++，it–），也支持指针移位操作（it+4）。

  • v.begin()获取vector的头指针，v.end()获取vector的尾地址的下一个地址（这个地址没有元素），在用迭代器遍历的时候，我习惯用

    for (auto it = vector1.begin(); it != vector1.end(); it++)  // 遍历数组

  • 在STL中只有vector和string允许用迭代器+整数的方式访问元素，因为其他的STL容器中元素的存储位置不一定是连续的

vector常用函数：

  1. push_back()：在vector末尾添加一个元素，时间O(1)

  2. pop_back()：弹出vector末尾元素，时间O(1)

  3. front()：返回vector的第一个元素值，但不弹出，若vector为空会报段错位

  4. back()：返回vector的最后一个元素值，但不弹出，若vector为空会报段错误

  5. size()：获取vector元素个数，时间O(1)，返回的是无符号整数

  6. clear()：清空vector，时间O(N)

  7. insert(迭代器it，元素值x)：在迭代器指向的位置插入元素x，时间O(N)

  8. erase(迭代器it)：删除it指向的值

  9. erase(迭代器first，迭代器last)：删除[first，last)间的值

  10. empty()：vector判空，空返回true，不空返回false

  11. rbegin()：返回指向尾部的指针，rend()：返回指向头部的前一个指针，依然用

    for (auto it = vector1.rbegin(); it != vector1.rend(); it++)  // 遍历数组

vector常见用途：

  • 存储不定长的数据
  • 邻接表存储图

vector与sort：

  1. 对于默认数据类型
  • 默认降序：sort(vector.begin(), vector.end()) == sort(vector.begin(), vector.end(), less<>())：这里千万要记得less<>后面还有一对括号()，记成vector的第一个元素是less（小的），所以是升序
  • 升序：sort(vector.begin(), vector.end(), greater<>())：这里千万要记得greater<>后面还有一对括号()，记成vector的第一个元素是greater（大的），所以是降序
  • 这个升序和降序的规则与优先队列是反着来的，优先队列里对于默认数据类型，less的话是大根堆（记成less是数组降序），greater是小根堆（记成greater是数组升序）
  1. 对于结构体类型
  • 在结构体内重载比较函数：只能重载小于号<：函数参数只有一个，比较函数里写<就是升序排列，写>就是降序排列，这个没有理解歧义
  • 在结构体外写比较函数：与结构体内一致，但不是重载函数，函数参数有两个，比较函数里写<就是升序排列，写>就是降序排列

2. set：内部自动有序（升序）且不含重复元素

#include<set> using namespace std;  // 头文件

声明：set s1；typename可以是基本数据类型和结构体和其他STL容器，若是STL容器，C++11之前要在末尾»中间加上空格；若是结构体，则必须在结构体内写好比较函数，因为set给元素自动排序，否则会有编译错误；map也一样

set s1[100]；意思跟上面vector一样

set只能通过迭代器访问：set s1::iterator it = s1.begin()；或者auto it = s1.begin()，同样都是it != s1.end()，取出元素用*it

set常用函数：

  1. insert(元素x)：插入一个元素，并自动排序，时间log(N)
  2. find(value)：查找元素，返回值为value的迭代器，未找到返回s1.end()，时间log(N)
  3. erase(迭代器it)：删除it指向的元素，可以结合find使用（erase(s1.find(100))），时间O(1)
  4. erase(value)：直接删除值，这个在基本数据类型中常用，时间O(logN)
  5. erase(first，last)：删除区间[first，last)
  6. size()：获取set内元素个数
  7. clear()：清空set
  8. empty()：判空
  9. count(int)：判断set中某个元素的个数，没有就返回0，常用来判断集合中是否存在某个元素
  10. lower_bound(int)：以二分查找方式在有序数列中找到第一个大于等于参数的值的迭代器，如果没找到就返回s1.end()
  11. upper_bound(int)：以二分查找方式在有序数列中找到第一个大于参数的值的迭代器，如果没找到就返回s1.end()

set常用来处理唯一的情况，若要处理不唯一的情况，用multiset（复集合），还有一种容器（C++11支持）unorder_set（无序set），用散列替代set的红黑树，只去重而不自动排序（结构体中可以不必写比较函数），时间比set快的多

3. string：封装好的字符串

#include<string> using namespace std;  // 头文件，（注意不是<string.h>，这个是字符串操作函数库）

声明：string s；string的输入只能用cin，输出可以用cout或者printf（s.c_str()）

string的访问：下标和迭代器，最常用的就是下标

  • 像数组一样的下标访问
  • 迭代器s.begin()，s.end()和上面用法都一样，支持自加自减操作和指针操作

string常用函数：

  1. +=：拼接字符串
  2. ==，!=，<，<=，>，>=：按照字典序比较大小，内部已经实现了，不必自己写
  3. length()/size()：返回字符串的长度，但是我常用length()
  4. insert(pos，string)：在pos位置插入string
  5. insert(it，it2，it3)：it表示源字符串的欲插入位置，it2和it3是待插字符串的首尾迭代器[it2，it3)，被插在it的位置上
  6. erase(it)：删除单个元素，it是需要删除元素的迭代器
  7. erase(first，second)：两个参数都是迭代器，删除[first，second）中间的字符串
  8. erase(int pos，int length)：删除从pos开始的length长度的字符串；若省略第二个参数，则是删除从pos位置开始一直到字符串末尾的子串，和substr一样
  9. clear()：清空字符串
  10. empty()：判空
  11. substr(int pos，int length)：取从pos开始的length长度的子串；若省略第二个参数，则是取从pos位置开始一直到字符串末尾的子串
  12. find(string str)：返回str在字符串中第一次出现的位置的下标，未匹配返回-1
  13. find(char c)：返回c第一次在s中出现的位置，未找到返回-1
  14. replace(int pos, int len, string str)：从pos开始，长度为length的子串替换为str
  15. replace(it，it2，str)：迭代器[it，it2)中的子串替换为str

4. map：映射容器，key自动升序，可将任何类型（包括STL容器）映射到任何类型（包括STL容器），甚至可以当数组用

#include<map> using namespace std;  // 头文件

声明：map<typename，typename> m；typename作为STL时候，C++11前»中要加空格；key作为结构体时，要提前写好比较函数

map的访问：通过key访问和迭代器访问

  • map中的key是唯一的，多次赋值会被覆盖
  • map<typename，typename>::iterator it；或者auto it，访问的时候it->first，表示key，it->second表示value（这里只能用指针）

注意点：

  • map中映射字符串只能用string，不能用char数组
  • 当写成map[key]的时候，就已经建立了一个映射，若是不给其赋值，value将按照默认值初始化，若value是结构体，则必须在结构体中写好默认构造函数，若key是结构体，则要提前写好比较函数

map常用函数：

  1. find(key)：返回键为key的迭代器，未找到返回map.end()
  2. erase(it)：it为需要删除的元素的迭代器，时间O(1)
  3. erase(key)：直接删除键为key的元素，时间O(logN)
  4. erase(first，last)：删除迭代器区间在[first，last)的元素
  5. size()：返回元素个数
  6. clear()：清空map
  7. empty()：判空

map常见用处：

  • 当做标记用
  • 当做唯一标识映射
  • 还有unorder_map（无序map），不需要写好比较函数，用散列替代map的红黑树，速度比map快的多

5. queue：队列，就是队列，比较简单

#include<queue> using namespace std;  // 头文件

声明：queue q；

queue的访问：queue并不能随时访问队列内所有的元素，只能访问队首和队尾的元素

  • 访问队首：front()；返回的是具体的元素，不是指针
  • 访问队尾：back()；同上

queue常用函数：

  1. push(data)：将元素入队，添加在队尾
  2. front()：返回队首元素，并不弹出
  3. back()：返回队尾元素，并不弹出
  4. pop()：弹出队首元素，没有返回值
  5. empty()：判空
  6. size()：返回队列元素个数
  7. queue没有clear()函数

queue常见用处：

  • 就是当队列用，但是在使用front和back的时候必须判empty
  • 常在BFS的时候使用

6. priority_queue：优先队列（堆），队首元素一定是优先级最高的那一个

#include<queue> using namespace std;  // 头文件，（对，没错在queue的头文件里，没有专门的优先队列的头文件）

声明：priority_queue pq；typename若是结构体，必须写好比较函数

priority_queue的访问：只能用过top()访问队首元素，也称为堆顶元素，就是优先级最高的元素（默认大根堆）

priority_queue常用函数：

  1. push(data)：元素入堆，同时调整堆结构，就是按照大小关系或者比较函数调整
  2. pop()：弹出堆顶元素，没有返回值
  3. top()：返回堆顶元素，但并不弹出
  4. empty()：判空
  5. size()：返回堆元素的个数
  6. 堆没有clear()函数

优先队列内元素优先级的设置：

  • 对于基本数据类型：优先队列默认按照降序排序（大根堆），下面两种写法是一样的：

    priority_queue<int> priority_queue1
priority_queue<int, vector<int>, less<> > priority_queue1  // 注意这里的less后没有()，与在sort中使用不同

    那么降序的话就是

    priority_queue<int, vector<int>, greater<> > priority_queue2 // 注意这里的greater后没有()，与在sort中使用不同

    这三个参数，第一个就是堆的参数类型，第二个是承载这个堆的容器，第三个是比较规则，记住就行了，我也不知道里面是什么东西

  • 对于结构体：只需要在重载比较函数的时候，在函数体内写好排序函数（重载<），但是规则反着来，就是return < ，是大根堆（反而是数组降序），return > ，是小根堆（反而是数组升序）

补充一点：在结构体里重载比较函数的时候（operator<），只能重载<，而不能重载>（会编译错误）。

再来一点：无论是写在结构体内还是外面的比较函数，其参数最好都用const加引用&，因为这样有安全效率又高

也有把比较函数写在外面的，在声明优先队列的时候加进去（其实没有写在结构体内舒服）

priority_queue<typename, vector<typename>, cmp> priority_queue1;

优先队列的常用处：

对一些算法进行效率上的优化，比如dijkstra算法，因为堆在排序问题和提取最大元素的问题上效率很高

在使用堆的时候，top()前要判empty()

7. stack：栈，就是栈，先进后出

#include<stack> using namespace std;  // 头文件

声明：stack s；

stack同样只能通过top()来访问栈顶元素，不能随时访问所有元素

stack常用函数：

  1. push(data)：元素入栈
  2. pop()：弹出元素，没有返回值
  3. top()：返回栈顶元素，并不弹出
  4. empty()：判空
  5. size()：返回元素个数
  6. stack也没有clear()函数

stack常见用途：

常用来模拟递归，当递归层数过多的时候，程序栈容易溢出，用stack可以避免这个问题，其实这样用的非常少

8. pair：过于简单就不想写了
